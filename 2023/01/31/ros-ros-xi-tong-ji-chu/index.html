ROS基础知识

工作空间

工作空间是一个文件夹workspace

一般在工作空间下有三个文件夹build src devel

src文件夹下放置的是用户的源代码，也就是功能包

build文件夹放置的是编译过程中生成的中间文件

devel文件夹放置的是编译生成的一些可执行文件

build和devel文件夹在使用ros时是可以忽略的

创建工作空间

1.创建文件夹

mkdir -p ~/new_workspace/src

mkdir 创建文件夹（linux）

-p检测这个路径 如果没有这个文件夹 就创建它

2.进入文件夹

cd new_workspace/src

3.将这个文件夹初始化成一个工作空间

catkin_init_workspace

编译工作空间

4.编译工作空间需要在src上一级目录下完成

cd .. 返回上一级目录

catkin_make再编译

5.编译完成后，在同一目录下生成了build和devel文件夹（还有一个.catkin_make_workspace文件）

6.但是catkin_make是编译全部工作空间，如果想编译单独的功能包，使用以下指令

catkin_make -DCATKIN_WHITELIST_PACKAGES="功能包名"

注意，此时的catkin_make也会直接编译指定的功能包

如果想解除，需要重新运行（将“”内改成空即可）

catkin_make -DCATKIN_WHITELIST_PACKAGES=""

7.还可以指定任务数和线程数编译

catkin_make -j4 -l4(4任务数4线程)

设置环境变量

编译完后需要添加环境变量，告诉系统这是工作空间的路径，这样运行工作空间内的程序才能被正确的索引到

8.进入workspace文件夹

source devel/setup.bash

9.检查有没有添加成功

echo SROS_PACKAGE_PATH

注意环境变量的有效性仅限于当前终端，再打开一个终端就不行了，因为此时添加的是局部环境变量

10.添加全局环境变量

nano .bashrc

在source栏再添加一行

source /home/Tonkla/new_workspace/devel/setup.bash

再source以下

source .bashrc

功能包

功能包是存放在工作空间中的src文件夹目录下的

一般代码文件除了主程序文件还有头文件、参数文件等

下图是一般功能包内常见文件类型

创建新功能包

进入src路径下

catkin_create_pkg <package_name> [depend1] [depend2] [depend3]

示例：catkin_create_pkg new_package std_msgs rospy rosc

后面三个是“依赖”

include里是.h的头文件

launch里是.launch可执行文件

map里是用来存放地图的文件夹

param和config文件夹基本都是.yaml的参数文件

scripts里是python文件

src内是.cpp文件

然后就是编译规则文件CMakeLists.txt和包描述文件package.xml

CMakeLists.txt

这个文件的作用是定义编译规则

具体可查CMakeList官方文档

常见宏（点开文件怎么读）：

package.xml

此文件是按照标签的形式来写的

package官方文档

launch文件

launch文件是一种可以实现多节点启动和参数配置的xml文件

下面列出launch文件标签总览

右边三个标签用的较少

根标签：

<launch>
    ...
</launch>

嵌套标签：


<include file="$(find 功能包名)/launch/include/base_serial.launch" /> 


<include file="$(find 功能包名)/launch/include/base_serial.launch.xml">
    ...
</include>

节点：ROS程序运行的单位是node

运行单个node的指令是rosrun：

rosrun <package_name> <executable|python>

node标签的含义就像rosrun命令一样，一个node标签就是启动一个节点，启动多个节点就是设置多个node标签

使用node标签代表rosrun：

<node pkg="package_name" type="executable|python" name="node_name" />

功能包名/可执行文件名/重命名的名字

示例：

rosrun turn_on_wheeltec_robot wheeltec_robot_node

<node pkg="turn_on_wheeltec_robot" type="wheeltec_robot_node" name="wheeltec_robot">

node标签除了上面三个属性以外，还有其他可选的属性

参数服务器

机器人系统中，参数众多，这些参数可能来自机器内部的变量，也可能来自外部的传感器，同时参数的传递也非常频繁，因此设置和修改参数的便捷性就显得十分重要。ROS中提供了参数服务器这个功能，用户可以在命令行/launch文件/C++/pyhton中去输入/读取/修改/加载参数服务器中的参数。简单来说，参数服务器是一个全局的“字典”

param是一种全局的添加，arg是在launch中的局部添加

ROS话题

话题通信是一种支持一对多和异步的通信机制，Publisher发布者将要发布的消息发布到话题上，Subscriber订阅者订阅消息（Topic）。（就像个人博客和粉丝）

ROS话题在终端的操作指令

一般先用rostopic list列举话题，再用rostopic echo /odom(话题名)打开话题消息（剩下的操作一般是另外开一个窗口来进行）

rostopic hz /odom是能够查看话题发布的频率

rostopic bw /odom能够查看当前话题的带宽

rostopic info /odom能够打印出话题的发布者和订阅者，以及话题的消息数据类型

可以看到odom这个话题的发布者是wheeltec_robot这个节点，订阅者是robot_pose_efk和nodelet_manager两个节点，消息数据类型是nav_msgs/0dometry(这里先把这个消息数据类型当成一个结构体，ROS官方定义了许多用于消息通信的结构体)

rostopic find是通过消息数据类型去查找一个话题（后面跟数据类型） rostopic type是根据话题显示消息数据类型（后面跟话题名）

rostopic pun后面跟话题名和消息结构（消息结构只需要按tab就可以了）

然后就可以在话题内修改信息，然后按回车

就会刷新一帧的话题信息（因为pub指令默认只发送一次）

如果加上-r 即rostopic pub -r 循环发送的速率
