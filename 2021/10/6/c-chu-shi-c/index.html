【C++】初识C++

发表于 2021-10-06 分类于 【C++】

本篇前言

C++入门篇，着重在C语言的基础上讲解C++的语法。C++版本目前已经更迭支C++20，但本系列仍然着重讲解C++11

C++入门

C++应用的常用领域

1.操作系统以及大型系统软件开发（阿里云开发→独立大型数据库）

2.服务器端开发（美团app→处理下单→每一单的处理→后端服务器的处理和数据的管理）

3.人工智能（底层算法：C/C++ 上层：python封装）

4.游戏开发（大型游戏对性能要求高）

5.嵌入式开发（与C结合 微型系统）

6.分布式系统（服务器上层)

7.物联网（扫地机器人、智能家居）

如何学好C++

C++最大的缺点，就是语法复杂，而且难学难精

第一个层次，C++基础（平平常常）

第二个层次，正确高效的使用C++（轻车熟路）

第三个层次，深入解读C++（出神入化）

第四个层次，研究C++（返璞归真）

第二层就要多从工作实战项目中吸取经验

C++进阶的好书很多，但是入门的好书很少

前期语法居多，中期联系居多，后期融会贯通居多

所以

前期一定要多写博客总结+练习熟悉语法

中期一定要多练习，学会运用

后期一定也要多写博客总结

C++共计63个关键字，C语言有32个关键字，多了几乎整整一倍。同时，C语言中的关键字C++都可以用，C++也涵盖了大部分C语言的语法

不逐个讲解，在语法中慢慢渗透学习

命名空间

为什么要提出“命名空间”？

C++每增加一个语法都是为了解决C语言做不到的东西

看下面的C程序：

#include<stdio.h>
int a = 0;
//int a = 0;
int main()
{
    int a = 1;
    return 0;
}

由此可见，C语言在不同作用域可以定义同名变量，但是同一作用域不行

#include<stdio.h>
#include<stdlib.h>
int rand = 10;
int main()
{
    printf("%d\n", rand);
    return 0;
}

由此可见，C语言中，定义变量名很可能会出现命名冲突的现象（和已有的变量重名、和函数名重名、和同伴开发的代码命名冲突 等等）

所以C++提出了命名空间来解决命名冲突的问题

命名空间的定义

#include
#include
namespace N1
{
    int rand = 10;
}
int main()
{
    printf("%p\n", rand);//rand是函数指针
    printf("%d\n", N1::rand);//N1中的rand是整型变量
    return 0;
}
