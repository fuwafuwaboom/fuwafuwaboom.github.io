【C语言从青铜到王者】第五篇·详解指针

发表于 2021-10-03 更新于 2021-09-27 分类于 【C语言从青铜到王者】

本篇前言

指针是C语言的灵魂，精通指针也就基本精通了C语言，希望我的这篇详解指针可以给学习C语言的朋友们一些启发。

指针是什么？内存是什么？

回答这两个问题之前，我们先思考一个基本的问题：编程的本质是什么？

我们可以感性的认为编程就是通过代码这种语言在和计算机进行沟通，向计算机描述问题，告诉计算机如何执行解决方案，从而借助计算机强大的运算能力帮助我们解决问题。我们学习编程就是要借助计算机这个强大的工具帮助我们解决现实中的问题。

所以就衍生出了两个基本问题：

一：如何向计算机描述一个问题？

二：如何告诉计算机一个合理的解决方案？

这两个问题，基本贯穿于我们学习计算机科学的始终。我们学习编程语言、学习计算机组成原理、学习数据结构和算法，学习相应的数学知识、学习建模，都是为了向计算机更好的描述问题，提供给计算机更优的问题解决方案。后来我们学习操作系统和网络，拓宽了计算机本身解决问题的能力，也是为了这两个目的。

那计算机是如何理解我们描述给它的问题的呢？

可以说，计算机是通过数据化处理来认识这个世界的。万事万物，在计算机眼里都是数据

比如说，有这样一个事实：“有一些苹果，它的单价是2元，现在我想获得3个苹果”

我们把苹果的价格设为一个变量apple_price，我需要支付的钱设为一个变量total_price那代码就可以写成：

int apple_price = 2;
int total_prince = 3 * apple_price;

int apple_price = 2;这句话用我们的“人话“来说，就是”苹果的单价是2元“，而对于计算机来说，就是”我在内存中开辟了一块4字节大小的空间，这个空间是专门划分给一个叫apple_price的变量的，而现在这块空间中按照整型变量的存放规则放入了十进制数字2“

用图像来表示就是：

所以这就可以回答”内存是什么“的问题了

内存就是一系列储存0或1的”格子“（途中的所有格子都是内存），每8个格子代表一个内存单元（一行格子），每开辟一块空间就是划定一段连续的格子（图中开辟int型变量的空间就是灰色格子），用来存放新的01序列，这些新的01序列就会根据特定的规则存放数据，比如int型数据就是需要32个格子，代表32位二进制序列，序列首元素代表符号，1正0负，其余的按照整数二进制规则表示数字

所以，计算机其实能读懂的就是这一连串的01序列，也叫机器语言，我们今后所写的所有程序，实际上都会被编译器翻译成无数的0和1储存在这些格子中，我们操作变量，修改代码，本质上也就是改变这些0和1的储存位置

而光是存放还不够，我们是需要继续通过编程操控这些0和1的，所以这些0和1必须整整齐齐的，按照一定顺序的摆放在格子里

所以继续思考一个问题：内存是如何有序的存放这一系列的0和1的呢？

这就引入了地址的概念：每个内存单元的编号就是地址

我们像上图一样给内存单元编号，第一个内存单元就是1号，第二个就是2号……但是问题是计算机有数以亿亿计的内存单元，而且计算机只认识二进制语言（0或1），所以我们是不能用十进制的数字来编号的

C语言规定：在32位系统上，地址是32位二进制序列，在64位系统上，地址是64位二进制序列

而32个0/1写出来过于繁琐，一般编译器展示给我们的序列都是8位16进制（64位系统就是16位16进制）的序列

也就是说，在32位系统上，每个地址是32位二进制序列，所以最多可以表征2^32个内存单元，一个内存单元的大小是一个字节，也就是1B，2^10字节就是1KB，2^20字节就是1MB，2^30字节就是1GB，2^32字节就是4GB，所以32位系统最多可以有4GB的内存

当然啦，随着科技的进步，我们对内存的需求远远超过了4GB。我们现在的电脑基本都是64位系统，也就是地址都是64位二进制序列，最多可以表征2^64个内存单元，也就是2^34GB的内存，这个数量，够我们人类继续用好几十年啦

理解了上文所说的以后，我们终于可以来解释指针是什么了

指针，也叫做指针变量，是具有指针类型的，专门用来存放地址的变量，大小在32位系统上为4字节，在64位系统上为8字节

通过操作指针，我们就可以很方便的操控和管理内存，这也是为什么C语言常常是底层编码语言的原因
