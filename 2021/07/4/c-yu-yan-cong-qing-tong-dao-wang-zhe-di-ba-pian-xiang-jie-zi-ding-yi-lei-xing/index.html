【C语言从青铜到王者】第八篇·详解自定义类型

Posted on 2021-07-05 Edited on 2021-11-07 In 【C语言】

本篇前言

结构体实际上是一种我们自定义的数据类型。结构体的出现使得我们可以给某个变量多个维度的数据。今天就让我们深入学习一下C语言的自定义类型：结构体、枚举体、联合体

结构体

数组是一组相同类型的元素的集合，结构也是一组相同类型的集合

区别是数组的类型只有一种，比如int、char、int*、double

而结构的类型是很多不同类型的集合，不同的结构可以代表不同的复杂对象，结构中的成员就是这个对象的不同的属性

结构体类型的声明

struct Stu
{
    char name[20];
    int age;
    float score;
};

Stu就是一个结构体，它实际上也是一种类型，和int类型、float类型一样是一种数据类型，但是它表征的不再是单一特征的数据，而是一个多维度的数据，比如一个学生（Stu）就包含了姓名（name）年龄（age）成绩（score）这三个维度的特征。

结构体变量的创建

全局结构体变量：

struct Stu
{
    char name[20];
    int age;
    float score;
}s1, s2;

局部结构体变量：

struct Stu
{
    char name[20];
    int age;
    float score;
};
int main()
{
    struct Stu s;
    return 0;
}

结构体的成员可以是另外一个结构体：

struct Stu
{
    char name[20];
    int age;
    float score;
};
struct Doc
{
    char name[20];
    int age;
    char subject[20];
};
struct class
{
    struct Stu wang;
    struct Stu li;
    struct Doc zhang;
};

结构体变量的初始化

struct Stu
{
    char name[20];
    int age;
    double score;
};
struct Doc
{
    char name[20];
    int age;
    char subject[20];
};
struct class
{
    struct Stu wang;
    struct Stu li;
    struct Doc zhang;
};
int main()
{
    struct class s = { {"王刚",18,95.5},{"李明",19,83.5},{"张哲",46,"Math"} };
    return 0;
}

结构体成员访问

1.直接访问

int main()
{
    struct class s = { {"王刚",18,95.5},{"李明",19,83.5},{"张哲",46,"Math"} };
    printf("%s", s.wang.name);
    return 0;
}

2.用结构体指针访问

int main()
{
    struct class s = { {"王刚",18,95.5},{"李明",19,83.5},{"张哲",46,"Math"} };
    struct class* ps = &s;
    printf("%s", ps->wang.name);
    return 0;
}

结构体传参

比如我们想写一个函数打印结构体的内容

写法一：直接传递结构体

#include<stdio.h>
struct Stu
{
    char name[20];
    int age;
    double score;
};
struct Doc
{
    char name[20];
    int age;
    char subject[20];
};
struct class
{
    struct Stu wang;
    struct Stu li;
    struct Doc zhang;
};
void print1(struct class t)
{
    printf("%s %d %lf\n%s %d %lf\n%s %d %s\n",
        t.wang.name, t.wang.age, t.wang.score,
        t.li.name, t.li.age, t.li.score,
        t.zhang.name, t.zhang.age, t.zhang.subject);
}
int main()
{
    struct class s = { {"王刚",18,95.5},{"李明",19,83.5},{"张哲",46,"Math"} };
    struct class* ps = &s;
    print1(s);
    return 0;
}

写法二：传递结构体的地址

#include<stdio.h>
struct Stu
{
    char name[20];
    int age;
    double score;
};
struct Doc
{
    char name[20];
    int age;
    char subject[20];
};
struct class
{
    struct Stu wang;
    struct Stu li;
    struct Doc zhang;
};
void print1(struct class t)
{
    printf("%s %d %lf\n%s %d %lf\n%s %d %s\n",
        t.wang.name, t.wang.age, t.wang.score,
        t.li.name, t.li.age, t.li.score,
        t.zhang.name, t.zhang.age, t.zhang.subject);
}void print2(struct class* pt)
{
    printf("%s %d %lf\n%s %d %lf\n%s %d %s\n",
        pt->wang.name, pt->wang.age, pt->wang.score,
        pt->li.name, pt->li.age, pt->li.score,
        pt->zhang.name, pt->zhang.age, pt->zhang.subject);
}
int main()
{
    struct class s = { {"王刚",18,95.5},{"李明",19,83.5},{"张哲",46,"Math"} };
    struct class* ps = &s;
    print1(s);
    print2(&s);
    return 0;
}

那如果非得选择一个方式，我们应该选择哪种方式呢？

大部分情况下我们应该优先选择传址调用

1.如果使用传值调用，是需要系统内存压栈的，我们需要临时开辟一块很大的空间，而且还得把数据一个一个的传过去，所以空间和时间上都有浪费，而传址调用仅仅需要传递一个地址，非常快速

2.传址调用可以直接改变结构体的值，而传值调用无法做到
