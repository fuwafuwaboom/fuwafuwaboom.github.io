向“C源码”学习代码规范

Posted on 2021-08-07 Edited on 2023-01-27 In 【编程基本功】

我们如何写出易于调试，有健壮性的代码呢？源码就是这样的代码，我们要多看多学习源码的写法来提升我们的代码风格

C语言中库函数的源码

逐步实现源码书写 my_strlen函数 

#include<stdio.h>
void my_strcpy(char* dest,char* src)
{
    while (*src != '\0')
    {
        *dest = *src;
        dest++;
        src++;
    }
    *dest = *src;
    return;
}
int main()
{
    char arr1[20] = "xxxxxxxxxx";
    char arr2[] = "hello";
    my_strcpy(arr1, arr2);
    printf("%s\n", arr1);
    return 0;
}

优化点一：把两个++放在等式两边，简化代码量

#include<stdio.h>
void my_strcpy(char* dest,char* src)
{
    while (*src != '\0')
    {
        *dest++ = *src++;
    }
    *dest = *src;
    return;
}
int main()
{
    char arr1[20] = "xxxxxxxxxx";
    char arr2[] = "hello";
    my_strcpy(arr1, arr2);
    printf("%s\n", arr1);
    return 0;
}

优化点二：把循环体设成循环条件，简化代码量

#include<stdio.h>
void my_strcpy(char* dest,char* src)
{
    while (*dest++ = *src++)
    {
        ;
    }
    return;
}
int main()
{
    char arr1[20] = "xxxxxxxxxx";
    char arr2[] = "hello";
    my_strcpy(arr1, arr2);
    printf("%s\n", arr1);
    return 0;
}

最后的版本就有点像C源码中string.h中strlen的参考写法

但是仍然和原码不同

assert断言

优化点三：如果函数接收的是空指针，就会出现解引用空指针的bug

怎样解决这个bug，就需要引入断言assert关键字

#include<stdio.h>
#include<assert.h>
void my_strcpy(char* dest,char* src)
{
    assert(src != NULL && dest != NULL);
    while (*dest++ = *src++)
    {
        ;
    }
    return;
}
int main()
{
    char arr1[20] = "xxxxxxxxxx";
    char arr2[] = "hello";
    my_strcpy(arr1, arr2);
    printf("%s\n", arr1);
    return 0;
}

断言assert的作用就是，如果assert（）中的表达式值为0，就会主动报错，从而帮助我们很好的定位代码的bug

所以断言就像一个预设的雷达，当触发断言，就会很容易的发现问题

const保证从属关系

优化点四：程序是把后一个字符串拷贝到前一个里，所以使用者会事先保证后一个字符串的长度一定小于等于前一个字符串

而如果使用者手滑，把两个字符串的顺序写反了怎么办呢？

我们可以把源字符串的形式参数指针src加一个const，保证我们无法通过这个指针修改指针指向的内存值

#include<stdio.h>
#include<assert.h>
void my_strcpy(char* dest,const char* src)
{
    assert(src != NULL && dest != NULL);
    while (*dest++ = *src++)
    {
        ;
    }
    return;
}
int main()
{
    char arr1[20] = "xxxxxxxxxx";
    char arr2[] = "hello";
    my_strcpy(arr1, arr2);
    printf("%s\n", arr1);
    return 0;
}

优化点五：为了完全贴近原码，我们再把函数返回值设置一下，函数的返回值就是目标空间的起始地址

#include<stdio.h>
#include<assert.h>
char* my_strcpy(char* dest, const char* src)
{
    assert(src != NULL && dest != NULL);
    char* ret = dest;
    while (*dest++ = *src++)
    {
        ;
    }
    return ret;
}
int main()
{
    char arr1[20] = "xxxxxxxxxx";
    char arr2[] = "hello";
    printf("%s\n", my_strcpy(arr1, arr2));
    return 0;
}

printf函数用了函数的链式访问

tips：打印字符串的起始地址，就是打印整个字符串

模拟源码

模拟源码风格实现my_strcpy函数

#include<stdio.h>
#include<assert.h>
char* my_strcpy(char* dest, const char* src)
{
    assert(src != NULL && dest != NULL);
    char* ret = dest;
    while (*dest++ = *src++)
    {
        ;
    }
    return ret;
}

模拟源码风格实现my_strlen函数

#include<stdio.h>
#include<assert.h>
int my_strlen(const char* str)
{
    assert(str != NULL);
    int i = 0;
    while (*str++)
        i++;
    return i;
}

由于字符串长度必然是自然数，所以返回值可以改成size_t，也就是无符号整型unsigned int

#include<stdio.h>
#include<assert.h>
size_t my_strlen(const char* str)
{
    assert(str != NULL);
    size_t i = 0;
    while (*str++)
        i++;
    return i;
}

在哪里查看源码

得到strlen的源码

size_t __cdecl strlen (const char * str)
{
        const char *eos = str;

        while( *eos++ ) ;

        return( eos - str - 1 );
}

return用的就是一个指针-指针的方式

strcpy的源码


char * __cdecl strcpy(char * dst, const char * src)
{
        char * cp = dst;

        while((*cp++ = *src++) != '\0')
                ;               /* Copy src over dst */

        return( dst );
}

“大师”的代码不会出错，所以没有加断言，但是我们是“学徒”，所以还是加上为好

_cdecl是函数调用约定：函数调用约定是一系列关于函数调用时编译器执行的细节规则
