先验框

以网格的中心为中心，存在先验框

先验框是我们事先在图上标好的，然后我们的工作就是将先验框调整到的大小。也就是说我们是先把图片分割成网格，再用每个网格自带的先验框进行调整

def.boxes:后面的数字就是每个网格对应多少个先验框，5776就是总共有多少个先验框

我们就对这5776个先验框分别进行调整，然后看5776个框中是否包含我们需要的物体，有的话就标出来。而我们碰到重合的先验框，就得通过一些得分算法来再次调整

命名空间

为什么要提出“命名空间”？

C++每增加一个语法都是为了解决C语言做不到的东西

看下面的C程序：

#include<stdio.h>
int a = 0;
//int a = 0;
int main()
{
    int a = 1;
    return 0;
}

由此可见，C语言在不同作用域可以定义同名变量，但是同一作用域不行

#include<stdio.h>
#include<stdlib.h>
int rand = 10;
int main()
{
    printf("%d\n", rand);
    return 0;
}

由此可见，C语言中，定义变量名很可能会出现命名冲突的现象（和已有的变量重名、和函数名重名、和同伴开发的代码命名冲突 等等）

所以C++提出了命名空间来解决命名冲突的问题

命名空间的定义

#include
#include
namespace N1 //定义了一个命名空间域
{
    int rand = 10;
}
int main()
{
    printf("%p\n", rand);//rand是函数指针
    printf("%d\n", N1::rand);//N1中的rand是整型变量
    //N1（域名）:: 变量 其中::就是
    return 0;
}

命名空间中可以定义 变量 函数 宏 类型

命名空间嵌套定义

#include
#include
namespace N1 
{
    int rand = 10;
    namespace N2//嵌套定义
    {
        int a = 1;
    }
}
int main()
{
    printf("%d\n", N1::N2::a);//a变量需要两次解耦
    return 0;
}

同名命名空间会自动合并

所以在定义同名命名空间的时候要注意定义不冲突

同一个工程中的同名命名空间可以跨文件自动合并

如何使用命名空间

三种方式：

1.全部展开到全局

using namespace 命名空间名

#include
#include
namespace N1 //定义了一个命名空间域
{
    int r = 10;
    namespace N2
    {
        int a = 1;
    }
}
using namespace N1;
int main()
{
    printf("%d\n", r);
    return 0;
}

所以“第一个C++程序”就是把std（C++标准库）这个命名空间全部展开：

#include
using namespace std;
int main()
{
    cout << "hello world" << endl;
    return 0;
}

优点是方便，但缺点是把自己的定义暴露了，容易造成命名污染

2.访问每个命名空间中的成员时，指定命名空间

命名空间名::成员名

#include
int main()
{
    std::cout << "hello world" << std::endl;
    return 0;
}

优点是不存在命名污染，缺点是用起来麻烦，每个都得指定命名空间

3.指定展开常用的成员，不常用的不展开

using 命名空间名::成员名

#include
using std::cout;
using std::cin;
using std::endl;
int main()
{
    cout << "hello world" << endl;
    return 0;
}

#include
#include
namespace N1 //定义了一个命名空间域
{
    int r = 10;
    namespace N2
    {
        int a = 1;
    }
}
using N1::N2::a;
int main()
{
    printf("%d\n", a);
    return 0;
}

3是折中1和2的解决方案，比较常用

IO流

输入输出流

#include//iostream是库头文件，不加.h，但是我们引用自己定义的头文件需要加.h
using namespace std;
int main()
{
    int n = 0;
    cin >> n;// >> 流提取运算符/输入运算符
    cout << n << endl;// << 流插入运算符/输出运算符
    return 0;
}

多次输入以空格和换行间隔

endl等价于输出了一个换行符'\n'

输出是从左向右输出

至于cin和cout到底是什么，在学习了类和对象以后再做解释

C++与C语言的输入输出方式的优劣与选择

C++的输入输出可以自动识别类型，这是比C语言方便的

#include
using namespace std;
int main()
{
    char ch = 'A';
    int i = 10;
    int* p = &i;
    double d = 3.1415926;
    cout << ch << endl;
    cout << i << endl;
    cout << p<< endl;
    cout << d << endl;
    return 0;
}

C++的小数默认只输出小数点后5位，不太方便，此时推荐配合printf使用

printf（“%.2f",a);表示小数点后两位

其实对于连续输出同类型的，比如结构体，使用C语言反而更方便，因为不用多写那么多<<符号

所以建议在C++中不用纠结于到底用哪一个，而是哪个好用用哪个

缺省参数

相对C语言，C++增加了缺省参数的概念用于补充函数的定义

在定义函数的时候直接指定形参的值，就是缺省参数

缺省参数仅在没有传参的时候起作用，一旦传参就没这个参数什么用了
