高级操作与案例分析

文件操作

什么是文件

电影 mp4文件

歌曲 mp3文件

图片 jpg文件

文本 txt文件

表格 xlsx文件

它们虽然格式不一样，但是它们的数据都是保存在硬盘上的，也就是存储器

存储器包含了内存和外存（硬盘）

我们所创建的变量，就是存储在内存中

文件，就是存储在硬盘中

内存与硬盘的区别：

1.内存空间更小，硬盘空间更大

2.内存访问更快，硬盘访问较慢

3.内存成本更高，硬盘更便宜

4.内存的数据易丢失，硬盘的数据断电仍然可以持久存储（但寿命一般是5-15年，并不是永久）

而硬盘上存储的数据就是以文件的方式来组织的

此电脑→C盘、D盘 这里的内容都是硬盘上的内容，也都是文件

而文件夹（目录）也是一种特殊的文件，它是一种目录文件

windows xp 我的电脑

windows7 计算机

windows8 这台电脑

windows10 此电脑

windows11 此电脑

。。。

但是即使都是文件，不同文件类型存储数据的形式都是不一样的，而且都具有非常复杂的存储算法

这里仅针对最简单的文本文件进行操作

文件路径

一般的文件都是存储在文件夹中，而文件夹又嵌套在另一个文件夹里，想找到具体文件，需要一串由文件夹名称组成的信息，这就是文件的路径

为了表示qq.exe这个文件的位置，我们就可以通过路径的方式来表示D:\program\qq\Bin\QQ.exe

每个文件对应的路径都是唯一的，所以路径也可以看成文件的标识

Windows支持正反/\两种斜杠，而linux和mac一般都是用正斜杠/，因为\一般都有特定含义，可能是转义字符，双反斜杠\\才代表反斜杠

打开文件

读写文件是文件操作最重要的功能，但是在读写文件前需要打开文件，读写完成后需要关闭文件。

# 使用open来打开一个文件
f = open('D:/Python环境/text.txt', 'r')
# 第一个参数表示文件路径 
# 第二个参数表示打开方式 r表示read 按照读方式打开 w表示write 按照写方式打开 a表示append 按照写方式打开，把内容写道原有文件内容的末尾
# f表示file是一种文件对象
print(f)
print(type(f))

第一行显示的是文件的基本信息

第二行是Python对这个文件类型起的名字

open函数的返回值是一个文件对象，文件对象是内存上的一个变量，而真正的文件是在硬盘上的。所以针对内存上的文件对象的操作都会被转换成对硬盘上文件的操作。所以此处的文件对象，就像一个“遥控器”，可以间接操作硬盘。计算机把这种远程操控的“遥控器”成为“句柄”（handler）

关闭文件

文件操作遵循：“有借有还，再借不难”

文件在打开使用完之后一定要关闭，打开文件其实是在申请一定的系统资源，整个文件操作过程都在依赖这些资源，不再使用这些资源时，就需要及时释放。也就是说申请的系统资源一定要及时释放，不然会造成文件资源泄露，可能会造成别的代码无法顺利打开。

关闭文件使用的是close方法

f.close()
# 关闭文件f

正是因为一个系统的资源是有限的，所以一个文件能够打开的文件个数也是有上限的

flist = []
count = 0
whilt True:
    f = open('d:/','r')
    flist.append(f)
    count += 1
    print(f'打开文件的个数：{count}')

上面的结果是2的整数次方再减3。因为每个程序在启动的时候，会默认打开三个特殊文件，它们不对应到磁盘：

1.标准输入 对应到键盘 input通过标准输入写数据

2.标准输出 对应到显示器 print函数就是往标准输出读数据

3.标准错误 对应到显示器

而在系统中也是可以通过一些设置项来配置打开文件的最大数目

使用close()就可以无限循环了

flist = []
count = 0
whilt True:
    f = open('d:/','r')
    flist.append(f)
    f.close()
    count += 1
    print(f'打开文件的个数：{count}')

为什么操作都必须把文件对象加入到列表中呢？因为如果不这么做，会看不到效果

因为Python中有垃圾回收机制(GC)，如果不继续使用文件，Python会自动释放文件

但是自动释放不一定及时，所以还是会有文件泄露的风险。所以还是要尽量进行手动释放，才能万无一失

写文件

# 使用write实现写文件操作
f = open('d:/Pyhton/test.txt', 'w')
f.write('hello')
f.close()

如果直接使用w方式打开，会清空文件原有的内容

# 使用append方式打开
f = open('d:/Pyhton/test.txt', 'a')
f.write('hello')
f.close()

使用追加写append就可以

读文件

读文件涉及到多种不同的操作

# 使用read方式读文件，可以指定读几个字符
f = open('d:/python/test.txt', 'r')
result = f.read(2)
print(result)
f.close()

会报错，因为：

中文和英文类似，在计算机中，都是用“数字”来表示字符的（ASCII码），但是哪个数字对应哪个汉字在计算机中有多个版本，中文主流有两个版本（GBK和UTF8）

txt文件的右下角会显示文件的编码方式

相比于gbk（一般仅限简体中文），utf8是使用更广泛的编码方式（全世界各种语言）

此处我们解决的方法还是让代码按照utf8来进行处理

# 使用read方式读文件，可以指定读几个字符
f = open('d:/python/test.txt', 'r', encoding = 'utf8')
result = f.read(2)
print(result)
f.close()

而更常见的需求的是按行来读取

# 最简单的方法，直接for循环，for循环默认按行读取
f = open('d:/python/test.txt','r', encoding = 'utf8')
for line in f:
    print(f'line = {line}')
f.close()

注意print会自动添加换行符，所以我们需要设置一下

print(f'line = {line}', end = '')

end表示每次打印需要在末尾加上东西，默认值是'\n'

# 还可以使用readlines方法直接把整个文件的所有内容都读出来，按照行组织到一个列表里
f = open('d:/python/test.txt','r', encoding = 'utf8')
lines = f.readlines()
print(lines)
f.close()

这种方式一鼓作气读完，所以这种方法速度更快

上下文管理器

我们写代码的时候还是很容易忘记关闭文件的

# 使用with语句
def func():
    with open('d:/python/test.txt', 'r', encoding = 'utf8') as f:
        #进行文件处理的逻辑

Part2：设置初始属性

颜值，体质，智力，家境 总和不超过20，每一项取值都是1-10之间

while True:
    print("请设置初始属性（可用点数总和为20）")
    face = int(input("请输入颜值（1-10）："))
    strong = int(input("请输入体质（1-10）："))
    iq = int(input("请输入智力（1-10）："))
    home = int(input("请输入家境（1-10）："))
    
    # 校验输入是否合法
    if face < 1 or face > 10:
        print('颜值输入有误')
        continue
    if strong < 1 or strong > 10:
        print('体质输入有误')
        continue
    if iq < 1 or iq > 10:
        print('智力输入有误')
        continue
    if home < 1 or home > 10:
        print('家境输入有误')
        continue
    if face + strong + iq + home > 20:
        print('总属性值超过20！')
    
    #输入都合法就结束循环
    print("初始属性输入完毕！")
    print(f"颜值：{face}，体质：{strong}，智力：{iq}，家境：{home}")
    break
